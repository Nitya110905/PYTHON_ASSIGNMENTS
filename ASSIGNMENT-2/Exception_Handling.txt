1. Explain Exception handling? What is an Error in Python? 

-> Exception handling is a mechanism in Python to manage runtime errors smoothly, preventing program from crashing. An Error is an issue that disrupts the normal flow of a program's execution.


2. How many except statements can a try-except block have? 

-> A try block can have one or more except statements. You can have specific handlers for different types of exceptions.


3. Name Some built-in exception classes:

-> ValueError, TypeError, IndexError, KeyError, FileNotFoundError, ZeroDivisionError


4. When will the else part of try-except-else be executed? 

-> The else block is executed only if the try block completes successfully without raising any exceptions.


5.  Can one block of except statements handle multiple exception? 

-> Yes. You can group multiple exceptions in a tuple within a single except block.


6. When is the finally block executed? 

-> The finally block is always executed, no matter what happens. It runs whether an exception occurred, was caught, or if the try block completed successfully.
